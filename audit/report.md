# PVE-defaults audit (issues #199 + #178)

Generated by `audit/main.go`. Two rules combined:
1. **Defaults rule (#199)**: Go zero value differs from the documented PVE default → send the wrong value when unset.
2. **Type rule (#178)**: PVE schema declares `"type": "boolean"` but the Go field is plain `int`/`uint`/`bool` instead of `IntOrBool` → wrong type on the wire.

Recommendation matrix: schema=bool + defaults match → `IntOrBool`; schema=bool + defaults differ → `*IntOrBool`; otherwise → `*T` only when defaults differ.

> **Note on `FirewallNodeOption.Enable` vs `FirewallVirtualMachineOption.Enable`**: per-node firewall defaults to `1` (ships enabled), per-VM defaults to `0` (opt-in). This asymmetry is intentional in PVE's three-gate firewall design (cluster + node + VM) and is *not* a schema bug.
